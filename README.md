# Weather Condition Predictor

An interactive Streamlit app that predicts weather conditions (Clear, Partly
Cloudy, Cloudy, Mist/Fog, Light Rain, Heavy Rain, Severe Weather) from live
meteorological readings, using Random Forest, XGBoost, KNN, and Logistic
Regression models trained on the
[Indian Weather Repository](https://www.kaggle.com/datasets/nelgiriyewithana/indian-weather-repository-daily-snapshot)
dataset (34,466 records, 543 Indian locations).

This is an extension of a B.Sc. Data Science capstone project. See the
"About" tab in the app for what changed and why.


## Run locally

```bash
# 1. Create a virtual environment (recommended)
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Train the models (only needed once, or after changing train_models.py)
python train_models.py

# 4. Launch the app
streamlit run app.py
```

The app will open at `http://localhost:8501`.

## Retraining

If you swap in an updated/larger dataset, just replace
`data/IndianWeatherRepository.csv` and re-run `python train_models.py` —
it regenerates every file in `models/` automatically. `app.py` doesn't
need any changes as long as the CSV keeps the same column names.

## Deploying for free (Streamlit Community Cloud)

1. Push this folder to a public (or private) GitHub repo.
2. **Important:** the `models/` folder must be committed too (or regenerate
   it via a build step) — Streamlit Cloud won't run `train_models.py` for
   you automatically. Easiest option: run `train_models.py` locally once,
   then `git add models/` before pushing.
3. Go to [share.streamlit.io](https://share.streamlit.io), sign in with
   GitHub, click "New app," point it at this repo and `app.py`.
4. Done — you'll get a public URL like
   `https://your-app-name.streamlit.app`.

## What's different from the original capstone report

| | Original report | This app |
|---|---|---|
| Target | Temperature cut into 9 arbitrary bins | Real weather condition categories |
| Class imbalance | Not addressed | `class_weight="balanced"`, macro F1 reported |
| Metrics | Accuracy only (misleading given imbalance) | Accuracy + macro precision/recall/F1 + confusion matrix |
| Interface | Jupyter notebook with `input()` prompts | Deployed interactive web app |
| Best model | XGBoost, 91% accuracy | XGBoost, 98.8% accuracy / 95.7% macro F1 |

## Known limitations

- **Nowcasting, not forecasting.** The app classifies the *current* weather
  condition from *current* readings (temperature, humidity, cloud cover,
  etc.) — it does not predict tomorrow's weather from today's. The dataset
  has only ~63 unique dates per location, not enough for reliable
  sequence-based forecasting.
- **Severe Weather is rare** (<1% of the dataset), so predictions for that
  class are less reliable — check the confusion matrix in the "Compare
  Models" tab.
- Cloud cover and precipitation are near-direct proxies for the condition
  label, which partly explains the high accuracy — a genuine forecasting
  system would need to predict without access to those same-moment readings.

## Ideas for further extension

- Add a genuine **forecasting** mode: use lag features (yesterday's readings)
  to predict tomorrow's condition, once more historical days are available.
- Add **SHAP** explainability plots per prediction.
- Add a live weather API (OpenWeatherMap, WeatherAPI) so users can pull
  real-time readings instead of manual sliders.
